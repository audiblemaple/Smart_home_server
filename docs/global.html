(async) addHotspot(req, res)

Asynchronously adds a new hotspot entry to the configuration file.
This function performs several key operations:
- Acquires a lock on the configuration file to prevent concurrent writes.
- Reads the existing configuration from the file.
- Generates a unique ID for the new hotspot based on the existing IDs.
- Adds the new hotspot data (received in the request body) to the configuration.
- Writes the updated configuration back to the file.
- Sends a response back to the client indicating the result of the operation.
- Releases the file lock.
Parameters:
Name Type Description
req Object The request object from Express.js, containing the new hotspot data in `req.body`.
res Object The response object from Express.js, used to send back the operation status.

(async) handleControlCommand(req, res)

This function reads an 'action' and 'node' from the request body. Based on the action ('open', 'close', 'stop'), it performs the corresponding operation. If the action is invalid, it sends a 400 response with an 'Invalid action' message. In case of successful execution of the action, it sends a confirmation message. Errors are caught and logged, and an 'Internal Server Error' message is sent.

(async) updateLight(nodeID, isOn)

Asynchronously updates the lighting configuration for a specified node. This function performs several key operations: - Validates the provided `nodeID` and `isOn` parameters. - Acquires a lock on the configuration file to prevent concurrent writes. - Reads and parses the existing configuration from the file. - Updates the 'isOn' status for the specified `nodeID` in the configuration. - Writes the updated configuration back to the file. - Logs the result of the operation to the console. - Releases the file lock.
Parameters:
Name Type Description
nodeID string The unique identifier for the node to be updated. This should be a non-empty string.
isOn boolean The new status to be set for the node. True to turn the node on, false to turn it off.
